Private Deployment & Operation Manual

To keep your team's progress completely private from the public web, this app utilizes a dual-repository Empty Shell security architecture. You will host the main UI code on a Public repository, but all sensitive progress data will be silently routed to an invisible Private repository using secure scopes.

PART 1: ADMINISTRATOR SETUP (YOU)

Follow these steps to deploy the application and establish your secure private backend.

01 Create the Public "App" Repository
  • Create a new Public repository on GitHub — e.g., AstraForge-App. (Optional: Select the MIT License during creation to allow others to freely fork and use the UI).
  • Upload ALL contents from your github-deploy folder into this repository. You must include the four HTML files (index.html, dashboard.html, devops_roadmap.html, AstraForge_Operation_Manual.html) AND the new css and js folders so the styles and logic load correctly!
  • Go to repository Settings -> Pages, set Source to Deploy from a branch.
  • Select the main branch and / (root) folder, then click Save.
  • Within 1–2 minutes, GitHub will give you a live website link. This is the UI shell!
02 Create the Private "Data" Repository
  • Create a second new repository, but this time select Private — e.g., AstraForge-Data.
  • Do not upload the HTML or CSS files here. Make sure it is initialized with at least a README.md file so the main branch is created.
  • For Weekly Automation: Upload the .github folder and the js folder into this Private Data repository. This allows the backend engine to calculate your weekly dashboard snapshots!
03 Invite Teammates to the Private Repo
Important: Only invite teammates to your Private repository (AstraForge-Data).
  • Go to your Private repository's Settings > Collaborators.
  • Click Add people and invite the GitHub accounts of any teammates.
04 Create Your Secure GitHub Token
  • Log in to GitHub and navigate via the UI to:
    Settings ➔ Developer settings ➔ Personal access tokens ➔ Fine-grained tokens and click Generate new token.
  • Set the Token name (e.g., "AstraForge Sync") and choose an Expiration date.
  • Under Repository access, choose Only select repositories and select your Private repository (AstraForge-Data).
  • In Repository permissions, locate Contents and select Read and write.
  • Click Generate token and copy the string immediately.
05 Launch the App
  • Visit your hosted public GitHub Pages URL from Step 1.
  • In the GitHub Repository section, enter your GitHub Username and the name of your Private Data Repository (e.g. AstraForge-Data).
  • Paste the Personal Access Token you generated in Step 4.
  • In the Your Profile section, define the JSON filename where your own progress will be saved (e.g., my_progress.json).
06 Add Teammates to Your Roster (OPTIONAL)
  • Scroll down to the Teammates section and click + Add Teammate to spawn a new row. (Up to 5 teammates.)
  • Under Name, enter your friend's display name. Click the glowing Color Dot to open the palette and pick a custom neon theme color — this color represents them across the entire dashboard.
  • Under Their Sync File in Repo, type the exact filename where their roadmap data is synced on GitHub — e.g., devops_roadmap_progress_p2.json. Must be in the same branch and repository as your main data.
  • Click the green Checkmark on the right of the row to verify the link. A pulsing green box means success — synced. A pulsing red box means error — double-check the filename, path, or your GitHub token.
  • Once your roster glows green, click Launch AstraForge to save your configuration and leap onto the unified dashboard!
Note: To remove a teammate at any point, use the dustbin icon next to their verification check.

PART 2: TEAMMATE SETUP (OTHER USERS)

Share these precise steps with your external teammates to allow them to securely sync their own progress files to the Private Tracker.

01 Accept the Private Repository Invitation
  • Wait for the Admin (Repository Owner) to invite you to the private data repository.
  • Check your email or visit your GitHub notifications to accept the invitation. You must accept this invitation before your token will work.
02 Create Your Secure GitHub Token
  • Log in to GitHub and navigate via the UI to:
    Settings ➔ Developer settings ➔ Personal access tokens ➔ Fine-grained tokens and click Generate new token.
    (Or go directly to: https://github.com/settings/personal-access-tokens/new)
  • Provide a Token name (e.g., "AstraForge Course Tracking") and choose an Expiration.
  • Scroll to Repository access and pick Only select repositories.
  • Search for the Admin's private repository (e.g. AstraForge-Data) and select it.
  • In Repository permissions, locate Contents and select Read and write.
  • Click Generate token at the bottom. Copy this string and do not lose it.
03 Configure Your Progress Session
  • Load the live dashboard web page public URL provided by the Admin.
  • Under GitHub Repository, enter the Admin's GitHub Username and the Admin's Private Repository Name (the same repo you were invited to).
  • In the Token field, paste your own newly generated Personal Access Token.
  • Under Your Profile, enter your name and write the exact Sync File name the Admin assigned to you (e.g., teammate_progress.json).
  • Click Launch AstraForge to initialize your private dashboard!
Security Note: Zero tracking data is transmitted to the public UI host. By generating the tokens specifically against the Private Data repository, you guarantee your local progress is written invisibly and exclusively to a secure, private destination under your team's complete control.

PART 3: DAILY INTELLIGENCE ACTIONS (OPTIONAL)

Your dashboard logic acts as a Daily Intelligence report, reading the exact day of the year and rotating through a new set of 4 Fame comments, 10 Mediocre comments, and 20 Shame comments for your team every single day.

To ensure the dashboard rigidly refreshes and invalidates GitHub's persistent HTML caches at exactly 7:00 AM IST every morning, deploy this precise GitHub Actions automation script.

01 Open Your Public Repository
  • Log in to GitHub and navigate to the Public App Repository (the one hosting your HTML files via GitHub Pages).
  • At the top of the repository, click the Actions tab.
  • If prompted, click set up a workflow yourself or New workflow.
02 Name the File
  • In the file name box at the top, name it exactly:
    .github/workflows/daily-refresh.yml
03 Paste the Pipeline Script
  • Delete any default placeholder text in the editor window.
  • Copy and paste the following strict YAML configuration:
name: Daily Dashboard Refresh (7am IST)
on:
  schedule:
    # 01:30 UTC is exactly 07:00 IST
    - cron: '30 1 * * *'
  workflow_dispatch: # Allows manual trigger

jobs:
  cache-bust:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pages: write
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v3

      - name: Force Cache Invalidation Update
        run: |
          date > cache_buster.txt
          git config --global user.name "GitHub Action Bot"
          git config --global user.email "actions@github.com"
          git add cache_buster.txt
          git commit -m "chore: Daily 7am IST cache bust" || echo "No changes to commit"
          git push
04 Commit the Workflow
  • Click the green Commit changes... button in the top right.
  • Leave the default commit message and click Commit changes directly to the main branch.
Success: Your repository will now execute this action 365 days a year. This script forcefully drops a daily timestamp file into the codebase at 7:00 AM IST, which successfully forces the GitHub Pages CDN to tear down its cache and rebuild the UI exactly at morning roll call.

PART 5: HIDDEN FEATURE (NO SETUP REQUIRED)

🤖 Intelligent Weekly Snapshots & Teammate Auto-Discovery (Beta)
Status: Active  |  Action Required: None

AstraForge includes a silent, automated backend engine powered by GitHub Actions. Every Monday morning at precisely 7:00 AM IST, your repository will briefly wake up to run the weekly-snapshot.js intelligence script entirely on its own.

What does it do?

  • Teammate Auto-Discovery: The engine scans your entire repository root and dynamically discovers any new teammate .json files you or your friends have added to the project. It intelligently infers their display names—no hardcoding or manual configuration required!
  • Progress Mathematics: It opens every single progress file, calculates how many tasks each person finished that week, and determines exactly which Phase they are currently battling.
  • Momentum Tracking: It compares everyone's current stats against the previous Monday to calculate their "weekly delta" (e.g., Uday generated +5% momentum this week!).
  • Permanent Logging: It compiles all these rankings and math into a highly structured data file and permanently archives it in your repository's /logs folder (e.g., logs/2026-W11.json).

Future Roadmap & Implementations

This automated data is quietly building a rich history of your team's performance. In future AstraForge updates, this historical data will automatically power features like:

  • Momentum Graphs: Beautiful line charts on the dashboard showing your team's velocity over the 6-month journey.
  • The "Crown" System: Automatic badges assigning the "Hustler of the Week" (highest % gained) and "Slacker of the Week" (lowest % gained) right on the multiplayer leaderboard.
  • Personalized Snark: Automated comments on your dashboard that analyze your personal logging history.
  • Time-to-Completion Predictions: AI-driven estimates calculating exactly what month you will finish the DevOps Roadmap based on your rolling 4-week moving average.
